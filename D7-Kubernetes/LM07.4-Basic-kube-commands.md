# Lab Manual: Run NGINX Pod and Deployment on Minikube Using CLI Only

## Objective

In this lab, you will:

1. Create a basic NGINX Pod using CLI commands only.
2. Expose the Pod using a Service.
3. Test the application.
4. Delete the Pod and Service.
5. Create an NGINX Deployment.
6. Expose the Deployment using a Service.
7. Scale the Deployment.
8. Verify and clean up resources.

---

# Part 1: Create a Basic NGINX Pod

## Step 1: Verify Minikube Status

```bash
minikube status
```

Expected:

* Host should be Running
* Kubelet should be Running
* APIServer should be Running

---

## Step 2: Create an NGINX Pod

```bash
kubectl run nginx-pod --image=nginx --restart=Never
```

Explanation:

* `kubectl run` creates a Pod.
* `nginx-pod` is the Pod name.
* `--image=nginx` uses the NGINX image.
* `--restart=Never` ensures only a Pod is created, not a Deployment.

---

## Step 3: Verify the Pod

```bash
kubectl get pods
```

Expected Output:

```bash
NAME        READY   STATUS    RESTARTS   AGE
nginx-pod   1/1     Running   0          1m
```

To get more details:

```bash
kubectl describe pod nginx-pod
```

---

## Step 4: Access the Pod Internally

Get Pod IP:

```bash
kubectl get pod nginx-pod -o wide
```

Example Output:

```bash
NAME        READY   STATUS    RESTARTS   AGE   IP            NODE
nginx-pod   1/1     Running   0          2m    10.244.0.10   minikube
```

You can also check the NGINX default page from inside the cluster:

```bash
kubectl exec -it nginx-pod -- curl localhost
```

If curl is not installed inside the container:

```bash
kubectl exec -it nginx-pod -- /bin/bash
apt update
apt install curl -y
curl localhost
exit
```

---

# Part 2: Expose the Pod Using a Service

## Step 5: Create a Service for the Pod

```bash
kubectl expose pod nginx-pod --type=NodePort --port=80 --name=nginx-pod-service
```

Explanation:

* `expose pod` creates a Service.
* `--type=NodePort` exposes the application externally.
* `--port=80` exposes port 80.
* `--name=nginx-pod-service` is the Service name.

---

## Step 6: Verify the Service

```bash
kubectl get svc
```

Expected Output:

```bash
NAME                 TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)        AGE
nginx-pod-service    NodePort    10.102.15.21    <none>        80:30080/TCP   1m
```

---

## Step 7: Access the Service

Get Minikube URL:

```bash
minikube service nginx-pod-service --url
```

Example Output:

```bash
http://127.0.0.1:52341
```

Open the URL in the browser.

You should see the default NGINX welcome page.

---

# Part 3: Delete the Pod and Service

## Step 8: Delete the Service

```bash
kubectl delete svc nginx-pod-service
```

## Step 9: Delete the Pod

```bash
kubectl delete pod nginx-pod
```

Verify:

```bash
kubectl get pods
kubectl get svc
```

---

# Part 4: Create an NGINX Deployment

## Step 10: Create a Deployment

```bash
kubectl create deployment nginx-deployment --image=nginx
```

---

## Step 11: Verify the Deployment

```bash
kubectl get deployments
```

Expected Output:

```bash
NAME                READY   UP-TO-DATE   AVAILABLE   AGE
nginx-deployment    1/1     1            1           1m
```

Check Pods created by the Deployment:

```bash
kubectl get pods
```

Check ReplicaSet:

```bash
kubectl get rs
```

---

## Step 12: Expose the Deployment

```bash
kubectl expose deployment nginx-deployment --type=NodePort --port=80 --target-port=80 --name=nginx-deployment-service
```

---

## Step 13: Verify the Service

```bash
kubectl get svc
```

---

## Step 14: Access the Deployment Service

```bash
minikube service nginx-deployment-service --url
```

Open the URL in the browser.

You should see the NGINX welcome page.

---

# Part 5: Scale the Deployment

## Step 15: Scale the Deployment to 3 Replicas

```bash
kubectl scale deployment nginx-deployment --replicas=3
```

Verify:

```bash
kubectl get deployments
kubectl get pods
```

Expected:

* 3 Pods should be running.

---

## Step 16: Watch the Pods

```bash
kubectl get pods -w
```

Press `Ctrl + C` to stop watching.

---

# Part 6: Cleanup

## Step 17: Delete the Service

```bash
kubectl delete svc nginx-deployment-service
```

## Step 18: Delete the Deployment

```bash
kubectl delete deployment nginx-deployment
```

Verify Cleanup:

```bash
kubectl get all
```
